If you are looking to reduce the cost of your it department, you have come to the right place. The Cost Reduction Principles (CRP) do just that.

The cost reduction principles are a collection of sound advice to reduce you software development costs. It is based on the three qualities that any software project should have. Quality decision making, quality source code and quality integration and infrastructure.

  • Quality decisions

    Which path is choosen can have a dramatic effect on the budget. An overly complex solution will evelate the costs significantly. But an overly simplisic solution prove to be useless later on. A balance must be sought.
    Explore Quality decisions

  • Quality code

    Source code is the hart and bones of a software project. How it is structured has a dramatice effect on the costs associated with a software project. Best practices must be applied.
    Discover Quality code

  • Quality integration

    Deployment, integration and infrastructure should not soak up lots of time from you engineers. Good engineers are rare and expensive so the overhead encountered by continous integration should be kept to a mimumum.
    Investigate Quality Integration
